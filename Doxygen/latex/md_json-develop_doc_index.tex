\end{DoxyItemize}\hypertarget{md_json-develop_doc_index_autotoc_md110}{}\doxysection{Container function overview}\label{md_json-develop_doc_index_autotoc_md110}
The container functions known from S\+TL have been extended to support the different value types from J\+S\+ON. However, not all functions can be applied to all value types. Note that the signature of some functions differ between the types; for instance, {\ttfamily at} may be called with either a string to address a key in an object or with an integer to address a value in an array.

Note that this table only lists those exceptions thrown due to the type. For instance, the \mbox{\hyperlink{classnlohmann_1_1basic__json_a93403e803947b86f4da2d1fb3345cf2c}{{\ttfamily at} }} function will always throw a \mbox{\hyperlink{classnlohmann_1_1basic__json_a4010e8e268fefd86da773c10318f2902}{{\ttfamily json\+::type\+\_\+error} }} exception when called for a string type. When called for an array, it {\itshape may} throw an \mbox{\hyperlink{classnlohmann_1_1basic__json_a28f7c2f087274a0012eb7a2333ee1580}{{\ttfamily json\+::out\+\_\+of\+\_\+range} }} exception if the passed index is invalid.
